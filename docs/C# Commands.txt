## .NET (backend/)

Build solution:
cd backend
dotnet build .\app.sln -c Release

Run WebAPI locally (no Docker):
- Requires SQL Server LocalDB (installed with Visual Studio / Build Tools) or change ConnectionStrings:DefaultConnection in SOH.WebAPI/appsettings.json to your instance.
- API listens on http://localhost:5130 (matches Flutter app). Pending EF migrations apply on startup.
- RabbitMQ is optional: appointment reminder publishing logs a warning if RabbitMQ is not running.

From repo root (PowerShell):
powershell -ExecutionPolicy Bypass -File .\scripts\backend-run-local.ps1

Or manually:
cd backend\SOH.WebAPI
$env:ASPNETCORE_ENVIRONMENT = "Development"
dotnet run --launch-profile http

EF migrations:
dotnet ef migrations add <Name> --project .\SOH.Services --startup-project .\SOH.WebAPI
